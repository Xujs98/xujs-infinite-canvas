# Docker 部署

主服务镜像由 `qq1371446705` 自行维护，发布到 Docker Hub：`qq1371446705/julong-canvas`。生产服务器通过 Docker Compose 运行，SQLite、提示词和上传素材统一持久化到宿主机 `./data`。

## Docker Hub 准备

公开仓库 `qq1371446705/julong-canvas` 已创建。你只需要在 Docker Hub 创建一个具备 Read/Write 权限的 Access Token，用于本地 `docker login`、构建和推送。

## 本地构建并推送（推荐）

先在本机登录 Docker Hub。密码提示中输入 Access Token，不要把 Token 直接写在命令、脚本、`.env` 或 Git 仓库中。

```bash
docker login --username qq1371446705
```

使用仓库脚本构建并推送。脚本默认同时发布 `linux/amd64`、`linux/arm64`，并给同一镜像写入指定版本和 `latest` 两个标签。

```bash
./scripts/docker-publish.sh v0.3.1
```

若 Buildx 只显示 `Dockerfile: RUN go build -o /server .`，使用下面的命令展开 Go 编译错误：

```bash
docker build --progress=plain --target api-build -t julong-canvas-api-debug .
```

等价的完整命令为：

```bash
docker buildx build \
  --platform linux/amd64,linux/arm64 \
  --tag qq1371446705/julong-canvas:v0.3.1 \
  --tag qq1371446705/julong-canvas:latest \
  --push \
  .
```

如果服务器确定是普通 Intel/AMD Linux，只发布 `amd64`：

```bash
DOCKER_PLATFORMS=linux/amd64 ./scripts/docker-publish.sh v0.3.1
```

发布后检查 Docker Hub 清单：

```bash
docker buildx imagetools inspect qq1371446705/julong-canvas:v0.3.1
```

## GitHub Actions 发布（可选）

在 GitHub 仓库 `Settings → Secrets and variables → Actions` 新增 Secret `DOCKERHUB_TOKEN`，值为 Docker Hub Access Token。推送 `v*` Tag，例如 `v0.3.1`，工作流会发布 `v0.3.1`、`sha-<commit>` 和 `latest` 多架构镜像。

## 使用发布镜像

```bash
git clone https://github.com/Xujs98/xujs-infinite-canvas.git
cd xujs-infinite-canvas
cp .env.example .env

# 必须修改管理员密码和 JWT 密钥；有域名时同时设置 PUBLIC_BASE_URL。
nano .env

mkdir -p data
docker compose pull
docker compose up -d
docker compose ps
```

服务器只拉镜像，不需要在服务器执行构建。需要部署固定版本时，把服务器 `.env` 中的 `IMAGE_TAG` 设置为已推送的版本，例如 `v0.3.1`，再执行：

```bash
docker compose pull
docker compose up -d --remove-orphans
```

启动后访问：

```text
http://服务器IP:3001
```

默认管理员账号：

```text
用户名：admin
密码：.env 中的 ADMIN_PASSWORD
```

## 本地构建镜像

如果需要基于当前源码构建镜像：

```bash
cp .env.example .env
docker compose -f docker-compose.local.yml up -d --build
```

## 日常升级

```bash
cd xujs-infinite-canvas
git pull --ff-only origin main
docker compose pull
docker compose up -d --remove-orphans
docker compose ps
docker compose logs --tail=100 app
```

固定维护流程为：本地执行 `./scripts/docker-publish.sh <版本>` 完成 build/push；服务器执行 `git pull --ff-only origin main`、`docker compose pull`、`docker compose up -d --remove-orphans`。服务器不负责编译镜像。

`docker-compose.yml` 默认使用 `latest`。建议正式发版后在 `.env` 中把 `IMAGE_TAG` 固定为 Git tag，例如 `v0.3.1`，保证部署可复现。

## 回滚

```bash
# 编辑 .env，把 IMAGE_TAG 改为上一个可用版本，例如 v0.3.0。
nano .env
docker compose pull
docker compose up -d --remove-orphans
docker compose ps
```

不要删除 `./data`，回滚容器不等于回滚数据库。涉及数据库结构变更时，应先恢复同版本的数据备份。

## 数据备份

```bash
mkdir -p backups
docker compose stop app
tar -czf "backups/julong-canvas-data-$(date +%Y%m%d-%H%M%S).tar.gz" data
docker compose start app
```

备份文件只包含持久化数据，不包含 `.env`。`.env` 含管理员密码、JWT 密钥和渠道密钥，应使用服务器的秘密管理或加密备份单独保存。

恢复前先停止容器，将当前 `data` 目录移走，再解压指定备份，确认目录权限后启动容器。

## 首次部署并迁移本地数据

需要把本地开发环境的用户、订阅、系统设置、SMTP、模型配置和素材一起迁移时，必须在本地 Go 服务停止写入后归档整个 `data` 目录。归档前执行 SQLite checkpoint 和完整性检查：

```bash
cd /Users/xujs/Downloads/xujs-infinite-canvas-main

LISTENER_PID="$(lsof -tiTCP:8080 -sTCP:LISTEN)"
PARENT_PID="$(ps -o ppid= -p "$LISTENER_PID" | tr -d ' ')"
kill "$PARENT_PID"

sqlite3 data/infinite-canvas.db 'PRAGMA wal_checkpoint(TRUNCATE); PRAGMA integrity_check;'
tar -czf "julong-canvas-data-$(date +%Y%m%d-%H%M%S).tar.gz" data
shasum -a 256 julong-canvas-data-*.tar.gz
```

将归档上传到服务器，在首次执行 `docker compose up` 之前恢复到仓库目录：

```bash
cd /opt/julong-canvas
tar -xzf /root/julong-canvas-data-YYYYmmdd-HHMMSS.tar.gz
test -s data/infinite-canvas.db
docker compose pull
docker compose up -d --remove-orphans
```

归档和服务器 `.env` 必须分开传输和保管。导入已有数据库后，`.env` 中的 `ADMIN_PASSWORD` 不会覆盖数据库内现有管理员密码；修改 `JWT_SECRET` 会让已有登录令牌失效，但不会删除用户数据。

## 文档站镜像

文档站位于 `docs/` 目录，按带服务端能力的 Next.js standalone 应用单独构建，不打进主应用镜像。


基于当前源码本地构建：

```bash
cd docs
docker compose -f docker-compose.local.yml up -d --build
```

## 数据目录

`docker-compose.yml` 会把本地 `./data` 挂载到容器内 `/app/data`，用于保存 SQLite 数据库、提示词数据和上传素材。

Docker 部署时建议把 `.env` 中的 SQLite 路径设置为：

```text
DATABASE_DSN=/app/data/infinite-canvas.db
```

如果需要让火山方舟拉取本地上传的 Seedance 参考素材，还需要把 `PUBLIC_BASE_URL` 设置为公网可访问的站点地址。

生产环境必须修改 `ADMIN_PASSWORD` 和 `JWT_SECRET`。`docker-compose.yml` 会在这两个变量缺失时拒绝启动，避免使用默认凭据上线。
